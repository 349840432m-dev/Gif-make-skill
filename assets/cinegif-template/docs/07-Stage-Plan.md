# CineGIF 视频转 GIF 文档同步阶段计划

## 1. 本阶段目标

在现有单图 GIF 和多图轮播文档基础上，同步第三种“视频转 GIF”模式：用户可导入一个本地短视频，选择 8 秒以内片段，在 240 帧预算内通过原生 `video` + Canvas 抽帧并导出 GIF。完成标准是单图、多图和视频三条主路径与失败路径均可按 01-06 文档执行与验收。

## 2. 已确认范围

- 技术栈：React + TypeScript + Vite + Canvas + `gifenc`。
- 模式：用户显式选择“单图动效 / 多图轮播 / 视频转 GIF”；素材数量或类型不得自动切换模式。
- 图片管理：追加、排序、删除。
- 视频管理：单视频导入、替换、元数据读取、片段范围和帧预算校验。
- 转场：直接切换、淡入淡出、横向滑动。
- 画布：项目内统一比例和尺寸，图片 `contain`，背景 `#0B0D12`。
- 视频：默认最大 100 MB、最长 8 秒片段、最多 240 帧；使用浏览器原生 `video` 元素和 Canvas，不引入 `ffmpeg.wasm`。
- 数据：仅当前浏览器内存，不上传、不持久化。
- 兼容：六种既有单图动效、参数和导出能力保持可用。

## 3. 明确不做

- 不实现音频、文字、滤镜、时间轴、关键帧或多轨。
- 不实现逐张时长、逐段转场、转场方向或单图动效叠加。
- 不实现视频裁切框、字幕、音频导出、多视频拼接、变速、倒放或自动格式转换。
- 不实现账号、云端服务、项目保存、分享和移动端专项优化。
- 不更换编码库，不预建 Worker 池，不新增运行时依赖，不引入 `ffmpeg.wasm`。
- 不修改本阶段文档范围之外的产品目标。

## 4. 实施阶段

### 阶段 A：模式契约与状态

做什么：

- 增加显式三模式、视频素材、视频片段设置和视频错误代码。
- 保持多图以稳定 ID 维护顺序，但模式不再由图片数派生。
- 保留现有单图设置和调用兼容层。

明确不做：

- 不在 React 组件中实现转场算法。
- 不实现自动模式切换。
- 不引入全局状态库。

验证：

- 0/1/2/10/11 张图片边界和视频 100 MB/8 秒/240 帧边界测试通过。
- 追加、移动、删除保持集合一致且不可变。
- 单图、多图与视频显式切换时各自设置不被覆盖。

### 阶段 B：批次导入与素材管理

做什么：

- 支持选择或拖入 1-10 张图片和后续追加。
- 实现批次数量/总大小原子校验及逐项解码结果。
- 实现拖拽排序、键盘前移/后移和删除资源释放。

明确不做：

- 不从 EXIF 或文件名自动排序。
- 不静默截断超额文件。

验证：

- 混合合法/非法文件时合法项按选择顺序加入，失败项可定位。
- 超过 10 张、60 MB 或 8000 万总像素时整次追加不改变原列表。
- 排序后预览顺序一致；删除只释放目标素材。

### 阶段 C：视频导入与片段校验

做什么：

- 支持导入单个 MP4、WebM 或 MOV，并通过原生 `video` 读取元数据。
- 实现 100 MB 文件上限、8 秒片段上限和 240 帧预算校验。
- 实现视频替换和对象 URL 释放。

明确不做：

- 不引入 `ffmpeg.wasm` 或服务端转码。
- 不读取、展示或导出音频轨。
- 不支持多个视频合并。

验证：

- 可解码视频读取出时长和分辨率，默认片段为前 8 秒以内。
- 超过 100 MB、片段超过 8 秒或帧数超过 240 时导出被阻止。
- 替换视频释放旧资源，旧导出结果过期。

### 阶段 D：统一画布与多图预览

做什么：

- 固化第一张有效图片的原始比例，建立项目唯一画布。
- 实现 `contain` 绘制和固定背景色。
- 实现多图时间映射及首尾循环。
- 视频模式使用视频宽高建立原始比例，并复用同一 `contain` 规则。

明确不做：

- 不提供裁切、智能构图或逐张画布。

验证：

- 横图、竖图、方图混合时不拉伸、不裁剪。
- 排序不改变已建立的原始画布比例。
- 2 张和 10 张均能按顺序循环，最后一张正确回到第一张。
- 视频帧不拉伸、不裁切，预览与导出比例一致。

### 阶段 E：三种转场

做什么：

- 用纯函数实现 `none`、`fade`、`slide` 绘制参数。
- 接入展示时长与转场时长校验。
- 预览配置变化后取消旧帧并从第一张重启。

明确不做：

- 不增加第四种转场，不提供逐段配置。

验证：

- 三种转场在 0、边界前、边界后进度的单元测试通过。
- 转场时长不小于展示时长时被入口拒绝。
- 三种转场首尾循环无空帧、透明闪烁或错误素材。

### 阶段 F：GIF 导出与并发安全

做什么：

- 让现有 `gifExporter` 使用统一帧提供器。
- 支持多图和视频总帧数、进度、取消、失败恢复和对象 URL 释放。
- 使用 `taskId` 阻止过期任务覆盖当前状态。

明确不做：

- 不并行运行多个导出任务，不在本阶段迁移 Worker。

验证：

- 1、2、10 张默认配置均生成有效循环 GIF。
- 8 秒以内视频片段在 240 帧预算内生成有效 GIF。
- 预览与导出画布、顺序和转场一致。
- 取消或失败后可再次生成，无残留编码任务或过期回调。

### 阶段 G：UI 收口与验收

做什么：

- 完成三模式入口、模式标签、素材条、视频片段控件、转场卡、错误列表和导出摘要。
- 补齐键盘排序、焦点、`aria-live` 和降低动态效果支持。
- 同步后续实现记录、测试验收和 Changelog。

明确不做：

- 不重做现有视觉体系或信息架构。

验证：

- 主要流程仅用键盘可完成，排序结果有文字反馈。
- 所有失败路径保留已有素材和有效设置。
- 当前稳定版 Chrome、Edge、Firefox、Safari 完成关键路径验收。

## 5. 失败与决策门

- 文档与现有代码冲突：优先保持单图兼容，记录差异后再实施，不静默改契约。
- 主线程导出阻塞：先验证让出事件循环和降低参数；仍不达标时单独记录 ADR 再决定 Worker。
- 浏览器解码差异：先使用 `HTMLImageElement` 回退；无法解码则拒绝该素材。
- 视频解码差异：只使用当前浏览器原生 `video` 能力；无法读取元数据、seek 或解码时拒绝该视频，不自动转码。
- 内存不足：停止导出并保留项目，提示降低尺寸、帧率、图片数或时长，不无限重试。
- 自动化检查失败：不得宣布阶段完成，记录失败命令和修复结果。

## 6. 验收矩阵

| 场景 | 预期 |
|---|---|
| 1 张图片 | 进入单图模式，六种动效可预览和导出 |
| 2 张图片 | 多图模式可预览和导出，默认淡入淡出 |
| 10 张图片 | 可排序、删除、预览和导出 |
| 第 11 张 | 明确拒绝，原列表不变 |
| 多图删除至 1 张 | 禁止导出并提示至少需要 2 张，可继续追加 |
| 横竖图混合 | 统一画布 `contain`，无拉伸和裁切 |
| 三种转场 | 预览与 GIF 一致，首尾正确循环 |
| 视频导入 | MP4/WebM/MOV 且小于等于 100 MB 时可读取元数据 |
| 视频片段 | 8 秒以内且小于等于 240 帧时可预览和导出 |
| 视频超限 | 超 100 MB、超 8 秒或超 240 帧时阻止导出并保留设置 |
| 视频解码失败 | 不清空当前设置，提示替换视频或缩短片段 |
| 取消/失败 | 保留项目，可重试，无旧任务覆盖 |
| 隐私 | 无图片、视频、Blob、路径、像素或视频帧上传和持久化 |
