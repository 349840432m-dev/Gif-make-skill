# CineGIF 产品设计

## 1. 设计原则

- 用户通过“单图动效 / 多图轮播 / 视频转 GIF”模式开关选择工作模式。
- 单图兼容：已有单图动效和参数保持原行为。
- 顺序可见：素材列表顺序始终等于预览和导出顺序。
- 统一画布：不同图片和视频帧共享输出比例和尺寸，预览即导出。
- 本地可信：界面持续说明素材仅在当前浏览器处理。
- 失败可恢复：单个素材或导出失败不清空其他素材和设置。

## 2. 信息架构

应用保持单页三步向导：

1. **导入素材**
2. **编辑效果**
3. **导出 GIF**

第二步按当前模式展示单图动效面板、多图轮播面板或视频片段面板；第三步共享导出流程。

## 3. 核心流程

### 3.1 模式选择与导入

- 用户先显式选择单图、多图或视频模式；素材数量和类型不得触发自动切换。
- 单图模式可点击或拖放 1 张图片；多图模式可点击或拖放 2-10 张图片；视频模式可点击或拖放 1 个视频。
- 系统按用户选择顺序逐张校验格式、大小、像素和解码结果。
- 校验成功的图片进入图片素材列表；视频校验成功后进入视频素材槽；失败项显示文件名和原因，不阻断其他合法项。
- 单图模式一次接收 1 张图片；多图模式接收 2-10 张图片并支持追加。
- 视频模式一次只接收 1 个 MP4、WebM 或 MOV 文件；再次导入为替换。
- 编辑阶段可继续追加，追加项排在列表末尾；若会超过 10 张，整次追加不执行。

### 3.2 单图模式

- 保持六种既有动效、实时循环预览及现有参数。
- 切换到多图后，单图参数保留在内存但不参与多图预览或导出。
- 切换到视频后，单图参数保留在内存但不参与视频抽帧或导出。
- 多图删除至 1 张时仍停留多图模式，禁止导出并提示继续追加图片。

### 3.3 多图模式

- 素材条展示缩略图、序号、删除按钮和拖拽手柄。
- 鼠标可拖拽排序；键盘聚焦缩略图后可使用“前移/后移”操作。
- 删除立即更新顺序和预览；删除最后一张需确认。
- 用户选择直接切换、淡入淡出或横向滑动，全部相邻图片及首尾循环使用同一转场。
- 用户调整单张展示时长和转场时长；非法组合在入口阻止并显示原因。
- 播放头按顺序循环；重排、删除、追加或改参数后从第一张重新预览。

### 3.4 视频转 GIF 模式

- 素材区展示视频文件名、大小、时长、分辨率和替换入口。
- 片段控件包含开始时间、结束时间和片段时长摘要；默认开始时间为 0 秒，最长 8 秒。
- 帧预算摘要实时显示 `预计帧数 / 240`，超出预算时禁用导出。
- 预览阶段直接播放原生 `video` 元素，并持续把当前播放帧绘制到 Canvas；导出阶段再按目标帧时间 seek 抽帧。
- 视频模式不展示单图动效、多图素材条或多图转场，避免误认为可以叠加。
- 视频音频不参与预览、导出或设置。

### 3.5 统一画布

- 第一张有效图片建立默认画布比例。
- 后续追加和排序不自动改变画布，避免结果因排序意外跳变。
- 视频模式默认使用视频解码尺寸建立原始画布比例。
- 用户主动切换比例时立即重绘全部图片或视频帧。
- 图片和视频帧采用 `contain` 居中完整显示，画布空白区使用固定深色背景。
- 缩略图可裁切用于列表展示，但主预览和导出不得裁切原图内容。

### 3.6 导出

- 展示输出宽度、比例、帧率、质量、预计帧数和预计总时长。
- 任何素材、顺序、转场或输出参数变化都会使旧导出结果标记为过期。
- 生成期间锁定所有会改变帧结果的操作，但允许取消。
- 成功后显示文件大小、输出尺寸和下载入口。

## 4. 状态模型

| 状态 | 条件 | 可用操作 |
|---|---|---|
| `empty` | 无有效图片 | 导入 |
| `single-ready` | 1 张有效图片 | 替换、追加、单图编辑 |
| `multi-ready` | 2-10 张有效图片 | 追加、排序、删除、多图编辑 |
| `video-ready` | 1 个有效视频 | 替换、选择片段、视频预览 |
| `preview-error` | 某素材或渲染失败 | 重试、删除问题素材 |
| `export-ready` | 当前模式配置有效 | 调输出参数、生成 |
| `exporting` | 编码任务运行中 | 取消 |
| `exported` | Blob 生成成功且未过期 | 下载、重新生成 |
| `error` | 可恢复操作失败 | 按错误类型重试 |

核心状态只保存在内存。每张素材拥有稳定 `id`，排序只调整 `id` 顺序，不重建成功解码的资源。

## 5. 参数规则

### 5.1 共享输出参数

- 输出宽度：480、640、960、1280 px，默认 640 px。
- 帧率：8、12、18、24 fps，默认 12 fps。
- 比例：原始、16:9、4:3、1:1、9:16。
- 质量沿用现有 GIF 导出取值与默认值。

### 5.2 单图参数

- 沿用现有强度、时长、速度、循环和六种动效规则。

### 5.3 多图参数

- 图片数：2-10。
- 展示时长：0.5-4 秒，默认 1.5 秒。
- 转场时长：0-1.5 秒，默认 0.4 秒，不能超过展示时长。
- 转场：`none`、`fade`、`slide`，默认 `fade`。
- 循环固定开启，不提供关闭开关。

### 5.4 视频参数

- 格式：MP4、WebM、MOV，必须由浏览器原生 `video` 元素可解码。
- 文件大小：默认不超过 100 MB。
- 片段时长：0.1-8 秒，默认最多取前 8 秒。
- 帧预算：`ceil(durationSeconds * fps) <= 240`。
- 输出不包含音频；视频帧使用共享输出宽度、比例、帧率和质量。

## 6. 状态变化规则

- 追加、删除、排序、替换、显式模式变化或参数变化：取消当前预览帧调度，清除未下载的过期导出结果并重启预览。
- 导出中执行上述操作：控件禁用；用户必须先取消导出。
- 删除素材：只释放被删除素材的解码资源和对象 URL。
- 替换视频：释放旧视频对象 URL 和元数据引用，保留视频模式的可复用输出参数。
- 重置：二次确认后释放全部素材、导出 Blob 和临时资源，回到 `empty`。

## 7. 失败路径

- 混合选择中部分文件失败：合法项加入列表，失败项集中列出；若全部失败则停留原状态。
- 超过数量或总大小上限：整次追加失败，原列表不变。
- 排序指令无效、拖拽中断：原子回滚到操作前顺序。
- 转场时长不小于展示时长：不接受新值，聚焦问题控件并说明约束。
- 素材在渲染中失效：暂停并定位问题项，不跳过素材生成不完整 GIF。
- 删除至 1 张：停留多图模式，禁止导出并给出“至少需要 2 张图片”的非阻塞提示。
- 视频无法读取元数据或解码失败：停留视频模式，保留已输入的片段设置，提示更换文件。
- 视频片段结束时间早于开始时间、超过 8 秒或超过 240 帧：不接受导出，定位到片段或帧率控件。
- 视频 seek 超时或抽帧失败：中止当前任务，保留视频与片段范围，允许缩短片段后重试。
- 导出取消或失败：保留素材、顺序、转场、片段范围和输出参数。
- 下载失败：复用当前 Blob 再次下载。

## 8. 明确不做

- 不设计音频、时间轴、关键帧、多轨或逐帧编辑。
- 不设计逐张时长、逐段转场、图片间空白段或转场方向配置。
- 不允许单图动效与多图转场组合。
- 不允许视频模式叠加单图动效、多图转场、多视频拼接或音频导出。
- 不设计云端作品库、分享链接、登录或会话恢复。
- 不自动裁切、智能构图或改变素材内容。
