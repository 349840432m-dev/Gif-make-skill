# CineGIF 测试与验收

## 1. 验收目标

验证 CineGIF 在目标桌面浏览器中能以本地优先方式完成单图动效、多图轮播和视频转 GIF 三步主流程，并能在输入异常、性能不足、取消和导出失败时保持可恢复状态。

## 2. 自动化测试范围

### 2.1 单元测试

- 文件校验：空文件、MIME、扩展名、尺寸、像素警戒线。
- 动效函数：六种 `EffectId`、`progress` 边界、强度映射、首尾连续性。
- 画布尺寸：横图、竖图、方图、禁止无意放大。
- 导出帧数：时长与 fps 组合。
- 状态转换：前进、返回、替换图片、取消、失败、重新生成。
- 资源清理：对象 URL、动画帧和临时 Canvas 引用释放。
- 视频导入：格式、空文件、100 MB 上限、元数据读取失败和对象 URL 释放。
- 视频抽帧：片段范围、8 秒上限、240 帧预算、seek 成功/失败和 contain 绘制。

### 2.2 集成测试

- 导入有效图片后进入第二步。
- 选择每种动效并改变参数，预览重启且设置保留。
- 导出进度、成功 Blob、下载入口和参数过期状态。
- `AbortSignal` 取消、编码异常后的恢复。

## 4. 浏览器与设备

- macOS / Windows 当前稳定版 Chrome。
- 当前稳定版 Edge、Firefox、Safari。
- 桌面视口重点：1280×720、1440×900、1920×1080。
- 900 px 以下只验证布局不溢出，不作为完整移动端验收。

## 5. 质量门槛

- 六种动效没有明显露边、首尾跳变或方向错误。
- 默认 640 px、12 fps、3 秒配置可成功生成。
- 导出期间页面可反馈进度，主线程无长期完全失去响应。
- 失败、取消或替换图片后无重复编码、动画循环或失效对象 URL。
- 图片内容和 Blob 不产生网络请求，不写入持久化存储。
- 主流程无阻断级控制台错误。

## 7. 失败路径验收

- 解码失败：错误原因清晰，状态未污染。
- Canvas 不可用：编辑入口被阻止，兼容说明可见。
- 预览失败：动画停止，参数保留，可重试。
- 导出内存不足：任务结束并释放资源，建议降低尺寸、帧率或时长。
- 长时间编码：允许用户取消，取消后可重试。
- 下载失败：不重复编码即可再次下载。
- 离开正在导出的页面：出现可用的离开提醒；浏览器不支持时不承诺拦截。

## 10. 明确不做

- 不验收账号、云端同步、服务端渲染和分享。
- 不验收音频、文字、滤镜、时间轴和多视频拼接。
- 不以移动端性能或触控体验作为首版发布门槛。
- 不要求 GIF 色彩、压缩率达到专业桌面编码软件水平。
- 不进行超出既定 20 MB / 40 MP 上限的稳定性承诺。
