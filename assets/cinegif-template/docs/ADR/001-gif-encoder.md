# ADR 001：选择 gifenc 进行浏览器 GIF 编码

## 状态

已采用，2026-06-15。

## 决策

使用 `gifenc@1.0.3`，逐帧读取 Canvas RGBA 数据、量化到 256 色并写入 GIF。首版在主线程编码，每帧完成后让出事件循环，并用 `AbortSignal` 在帧边界取消。

## 理由

- 纯 JavaScript、体积小、MIT 许可证。
- API 直接支持逐帧 palette、延迟和循环。
- 不要求 Worker 脚本、WASM 或远程服务，符合本地优先和依赖克制。
- 可报告帧级进度，并在帧边界可靠取消。

## 替代方案

- `gif.js`：内置 Worker，但体积和 Worker 资源部署更复杂。
- 自建编码器：工作量和正确性风险明显更高。
- 服务端编码：违反图片不上传的核心边界。

## 取舍

- `gifenc` 不提供抖动，照片渐变可能有色带。
- 首版主线程量化会占用 CPU；通过输出上限、逐帧让出事件循环和取消控制降低影响。
- 若真实浏览器验收显示交互延迟不可接受，再将量化和编码迁入 Worker；不提前增加并发复杂度。
