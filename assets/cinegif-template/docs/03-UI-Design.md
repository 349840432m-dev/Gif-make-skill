# CineGIF UI 设计

## 1. 视觉方向

沿用现有中文深色单页界面，保持实时画布为视觉重心。顶部右侧承载模式选择，左侧窄栏线性展示三步流程，中间控制区按当前流程展示对应功能区，右侧展示实时画布。

### 1.1 视觉变量

- 页面背景：`#0B0D12`
- 面板背景：`#141821`
- 次级面板：`#1B2130`
- 主文字：`#F4F7FB`
- 次文字：`#98A2B3`
- 主色：`#7C5CFC`
- 成功：`#32D583`
- 警告：`#FDB022`
- 错误：`#F97066`
- 边框：`#2A3242`
- 统一画布留白：`#0B0D12`

颜色通过 CSS 变量定义；错误、选中、排序位置和步骤状态同时使用文字、图标或序号表达。

## 2. 页面骨架

- 顶栏：左侧为 CineGIF 标识，右侧为“单图动效 / 多图轮播 / 视频转 GIF”模式选择和“清空重置”按钮。
- 流程栏：位于主工作区左侧窄栏，纵向线性显示 `1 导入素材`、`2 编辑效果`、`3 导出 GIF`。
- 主工作区：左侧流程栏、中间功能区、右侧实时画布三栏排列。
- 中间功能区：流程 1 显示当前模式的素材导入；流程 2 只显示当前模式的专属操作或说明，例如单图动效选择；流程 3 显示导出、取消、重试和下载。
- 实时画布下方：常态展示当前模式的全局预览/导出参数，例如播放时长、帧率、画面比例、输出宽度、循环播放，以及视频片段开始/结束设置；输出宽度包含“原素材宽度”动态选项，用于降低默认尺寸压缩。
- 多图素材条：在流程 1 的中间功能区中显示，保持顺序可见。
- 视频片段设置：固定在实时画布下方的全局参数区中显示开始、结束、时长和帧预算。

宽度不足 900 px 时上下排列；首版不承诺手机窄屏完整编辑体验。

## 3. 导入界面

- 空状态先显示三种模式入口：“单图动效”“多图轮播”“视频转 GIF”。
- 单图/多图辅助文案明确支持 JPEG、PNG、WebP 和本地处理。
- 视频辅助文案明确支持 MP4、WebM、MOV，默认最大 100 MB、最长 8 秒片段。
- 导入成功后显示模式标签：“单图动效”“多图轮播 · N 张”或“视频转 GIF”。
- 部分失败时，在导入区下方显示持久错误列表；每项包含文件名、失败原因和关闭入口。
- 追加入口在素材列表末尾显示为“添加图片”，达到 10 张后禁用并说明上限。
- 视频模式不显示“添加图片”；替换入口文案为“替换视频”。

## 4. 单图编辑界面

- 保持现有 Canvas、六种动效卡片和参数控件。
- 素材区显示单张缩略图、替换、追加和删除。
- 追加第 2 张前必须由用户切换到多图模式；单图模式内追加只用于替换单图或提示切换。

## 5. 多图编辑界面

### 5.1 素材条

- 每项显示缩略图、从 1 开始的序号、文件名截断、拖拽手柄和删除按钮。
- 当前播放项使用主色边框并显示“正在预览”，不只依赖颜色。
- 拖拽时显示明确插入线；放置成功后序号同步更新。
- 键盘用户可聚焦图片项，使用可见的“前移”“后移”按钮完成排序。
- 删除按钮具有包含序号或文件名的可访问名称。

### 5.2 转场与节奏

- 三种转场选项：直接切换、淡入淡出、横向滑动。
- 每张卡片包含名称和一句效果说明；选中态使用边框、背景和对勾。
- 控件包括单张展示时长、转场时长；非法组合在控件下方就地提示。
- 多图模式隐藏单图六种动效和强度、速度控件，避免误认为可叠加。

### 5.3 统一画布

- 画布容器按当前输出比例显示，图片居中 `contain`。
- 图片未覆盖区域显示固定深色背景，不使用透明棋盘表示最终结果。
- 横竖图切换时画布外框不改变，只切换内部内容。
- 播放控制显示当前序号、总数、当前时间和总时长。

## 6. 视频编辑界面

- 控制区显示视频文件名、大小、原始时长、分辨率和“替换视频”。
- 片段范围提供轻量双端点时间轴和开始/结束时间两个精确输入；拖动端点时预览跳到对应视频时间节点。
- 视频参数提供播放速度控件，用于快放或慢放；帧率只控制顺滑度，不替代播放速度。
- 帧预算摘要显示“预计 N / 240 帧”；超出预算时使用警告色和文字说明。
- 预览区使用同一 Canvas 展示当前视频帧，视频元素可隐藏但必须保留可恢复错误提示。
- 视频模式隐藏单图动效卡、多图素材条和多图转场卡。
- 视频错误卡包含“重试读取”“替换视频”“缩短片段”三类操作。

## 7. 导出界面

- 单图、多图和视频共用输出宽度、比例、帧率、质量控件。
- 多图额外显示图片数、转场、预计总时长和预计帧数摘要。
- 视频额外显示片段开始、结束、片段时长和预计帧数摘要。
- 主按钮状态：“生成 GIF”“准备帧”“编码中 N%”“重新生成”。
- 成功卡片显示 GIF 大小、输出尺寸、总时长和“下载 GIF”。
- 设置或素材变化后，成功卡片显示“结果已过期”，下载按钮禁用直至重新生成。

## 8. 组件状态与可访问性

按钮、转场卡、素材项、上传区和输入控件至少覆盖默认、悬停、键盘焦点、选中、禁用、加载和错误状态。

- 焦点环使用 2 px 主色外描边。
- 排序后使用 `aria-live` 宣布“图片 N 已移动到第 M 位”。
- 删除、显式模式切换和导出失败使用文本反馈。
- 视频开始/结束时间输入需要可键盘编辑，并在非法范围时把错误文本关联到对应控件。
- `prefers-reduced-motion` 关闭界面装饰动画，但不关闭用户主动播放的 GIF 内容预览。

## 9. 失败路径展示

- 单个文件失败：在素材区持久显示，不用短暂 Toast。
- 超过 10 张或总大小超限：在追加入口附近显示错误，原列表不闪动。
- 视频超过 100 MB、格式不支持或浏览器无法解码：在视频导入区显示持久错误，当前视频状态不被清空。
- 视频片段超过 8 秒或 240 帧：在片段控件和导出按钮附近同时提示，导出按钮禁用。
- 视频 seek/抽帧失败：Canvas 显示错误层，提供重试和缩短片段建议。
- 排序失败：素材条恢复原顺序，并在其上方显示恢复提示。
- 图片渲染失败：Canvas 覆盖错误层，同时高亮对应素材，提供“重试”和“删除该图”。
- 删除最后一张：确认框明确说明将清空当前项目。
- Canvas 不支持：显示不可关闭的兼容提示并隐藏编辑、导出操作。
- 导出失败：进度区切换为错误卡片，保留“重试”和降级建议。
- 下载被阻止：成功卡片保持，按钮改为“再次下载”。

## 10. 明确不做

- 不新增多轨时间轴、逐帧面板或独立模式选择页。
- 不提供逐张设置弹窗、裁切编辑器或智能构图。
- 不为三种转场制作视频缩略图，可使用轻量实时或静态示意。
- 不提供视频裁切框、音频波形、字幕轨、多视频列表或 `ffmpeg.wasm` 进度面板。
- 不提供浅色主题、移动端专属手势、登录、作品库或社交分享界面。
