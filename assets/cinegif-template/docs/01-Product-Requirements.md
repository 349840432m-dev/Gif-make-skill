# CineGIF 产品需求

## 1. 产品定义

CineGIF 是一个本地优先的 GIF 生成器。用户可导入一张图片制作单图动效、导入 2-10 张图片制作轮播 GIF，或导入一段本地短视频截取片段转为 GIF；图片解码、视频抽帧、Canvas 渲染、GIF 编码与下载均在浏览器内完成。

本阶段目标是在不破坏现有单图和多图口径的前提下，同步最小可用的视频转 GIF 文档链路。

## 2. 目标用户与场景

- 内容创作者：将封面、海报、摄影作品制作成循环动图。
- 运营与设计人员：把一组产品图、活动图或步骤图快速制作为轮播 GIF。
- 视频创作者：从本地短视频中截取 8 秒以内片段制作轻量 GIF。
- 隐私敏感用户：希望素材不上传、不进入云端处理。

核心成功场景：

- 单图：导入 1 张图片，选择已有动效并导出 GIF。
- 多图：导入 2-10 张图片，调整顺序和转场并导出循环 GIF。
- 视频：导入 1 个本地视频，选择片段范围并导出 GIF。

## 3. 产品目标

- 支持“单图动效 / 多图轮播 / 视频转 GIF”三种模式，由用户显式选择。
- 保持现有六种单图动效、参数和导出行为兼容。
- 支持 2-10 张图片的追加、排序和删除。
- 提供直接切换、淡入淡出、横向滑动三种多图转场。
- 支持本地视频文件的片段选择、帧预算校验和 GIF 导出。
- 将不同尺寸和比例的图片绘制到统一画布，预览与导出结果一致。
- 对输入、性能、资源和导出失败给出明确且可恢复的提示。

## 4. 功能范围

### 4.1 三种模式

| 用户选择 | 有效素材 | 可编辑能力 |
|---|---|---|
| 单图动效 | 1 张图片 | 六种现有动效及现有参数 |
| 多图轮播 | 2-10 张图片 | 排序、删除、追加、三种转场 |
| 视频转 GIF | 1 个视频 | 片段范围、输出参数、抽帧预览 |

- 模式由用户显式选择；导入素材数量或类型不得自动切换模式。
- 多图少于 2 张时保留在多图模式但禁止导出，并显示需要至少 2 张图片。
- 单图、多图和视频各自保留当前素材和设置，切换模式不自动改写另一模式。
- 图片顺序即轮播和导出顺序，最后一张使用同一转场回到第一张。
- 视频模式只接受单个视频素材；再次导入视频视为替换，不能与单图或多图素材混合导出。

### 4.2 三步流程

1. 导入素材：按当前模式选择或拖放图片/视频。
2. 编辑效果：单图选择动效；多图管理图片顺序并选择转场；视频选择片段范围。
3. 导出 GIF：选择输出尺寸、帧率和质量，生成并下载循环 GIF。

### 4.3 单图模式

保留电影推进、轻盈漂浮、呼吸脉动、左右摇曳、环绕镜头、景深视差六种动效，以及现有强度、时长、速度、比例、帧率、输出宽度和循环设置。既有单图输入、预览和导出调用方式不得因多图功能失效。

### 4.4 多图模式

- 图片数量：最少 2 张，最多 10 张。
- 图片管理：支持追加、拖拽排序、键盘排序和逐张删除。
- 转场：直接切换、淡入淡出、横向滑动；同一导出任务使用一个全局转场。
- 节奏：每张图片展示时长 0.5-4 秒，默认 1.5 秒；转场时长 0-1.5 秒，默认 0.4 秒，且不能超过单张展示时长。
- 循环：固定循环，最后一张平滑转回第一张。
- 本阶段不提供逐张时长、逐段转场或单图动效与多图转场叠加。

### 4.5 视频转 GIF 模式

- 输入：单个本地 MP4、WebM 或 MOV 视频文件，需由浏览器原生 `video` 元素成功解码。
- 文件上限：默认 100 MB；超过上限直接拒绝，不尝试压缩或分块处理。
- 片段范围：用户选择开始时间和结束时间，默认从 0 秒开始，最长 8 秒；结束时间必须大于开始时间。
- 帧预算：`ceil((end - start) * fps)` 不得超过 240 帧；超出时提示降低片段时长或帧率。
- 抽帧方式：使用原生 `video` 定位帧并绘制到 Canvas，再交给现有 GIF 编码流程。
- 循环：导出的 GIF 默认循环；不保留视频音频。
- 本阶段不提供音轨、字幕、裁切、逐帧编辑、变速、倒放或多视频拼接。

### 4.6 统一画布

- 整个项目只使用一个输出宽高比和输出尺寸。
- 默认沿用第一张有效图片的宽高比；用户仍可选择原始、16:9、4:3、1:1、9:16。
- “原始”在多图模式中明确指第一张有效图片的宽高比；排序不改变已建立的画布比例。
- “原始”在视频模式中指视频解码后的宽高比。
- 每张图片或视频帧使用 `contain` 完整放入画布，剩余区域使用深色背景 `#0B0D12`，不裁剪、不拉伸。
- 预览与导出必须使用同一画布尺寸、适配规则和转场计算。

## 5. 输入、输出与限制

- 输入：单图/多图为 1-10 张可解码的 JPEG、PNG 或 WebP；视频为 1 个可由浏览器解码的 MP4、WebM 或 MOV。
- 单文件上限：20 MB、4000 万像素。
- 多图总上限：10 张、60 MB、8000 万像素；逐张校验，任一失败时不加入素材列表。
- 视频文件上限：默认 100 MB；视频片段最长 8 秒；导出总帧数不能超过 240 帧。
- 输出：浏览器生成并下载的循环 GIF。
- 默认输出宽度为 1280 px；上传素材后可选择“原素材宽度”以减少尺寸压缩，也可选择 480、640、960、1280 或 1920 px；最高 24 fps。
- GIF 格式本身限制为最多 256 色，照片和渐变可能仍有色带或颜色压缩；本阶段不承诺无损输出。
- 多图预计总时长为 `图片数 × 单张展示时长`；导出总帧数不能超过 240 帧，超出时明确提示降低停留时间、帧率或图片数量。
- 所有素材和生成结果只驻留当前浏览器会话内存，不自动持久化。

## 6. 非功能要求

- 技术基线：React、TypeScript、Vite、Canvas、原生 `video` 元素、`gifenc`。
- 本地优先：应用资源加载后，核心流程断网可用。
- 兼容目标：当前稳定版 Chrome、Edge、Firefox、Safari。
- 可访问性：主要操作可键盘到达；排序、选中、错误和进度不只依赖颜色表达。
- 兼容性：现有单图成功路径、默认值和输出能力必须保留。
- 隐私：不上传图片、视频或 GIF，不记录文件内容、绝对路径、像素数据、视频帧或可识别隐私。

## 7. 失败路径

- 空文件、格式不支持或解码失败：拒绝该文件并说明原因，已成功加入的图片不受影响。
- 一次选择超过 10 张或追加后超过上限：不截断、不静默忽略，拒绝本次超额选择并提示剩余可添加数量。
- 单文件或总大小超限：拒绝对应操作，保留已有列表和顺序。
- 视频格式不支持、元数据读取失败或浏览器无法解码：拒绝该视频，保留当前视频模式设置并提示换用 MP4/WebM/MOV 或更短文件。
- 视频片段范围非法、超过 8 秒或超过 240 帧：禁止导出，聚焦片段或帧率控件并说明约束。
- 视频 seek 或抽帧失败：终止预览/导出，保留视频和片段设置，允许重试或缩短片段。
- 排序失败或拖拽取消：维持原顺序，不产生部分更新。
- 多图删除后只剩 1 张：停留多图模式并禁止导出，提示继续追加或手动切换到单图模式；删除最后一张需二次确认并回到空状态。
- 某张图片预览渲染失败：标记具体序号，暂停预览，允许删除该图或重试解码。
- Canvas 不可用：阻止编辑和导出，显示浏览器兼容提示。
- GIF 编码取消、超时、内存不足或失败：终止任务，保留素材、顺序和设置，建议降低尺寸、帧率、图片数或时长后重试。
- 下载被浏览器阻止：保留生成 Blob，提供再次下载，不重新编码。

## 8. 验收标准

- 导入 1 张图片时，现有六种单图动效均可预览和导出。
- 在多图模式导入 2、10 张图片时可预览和导出；第 11 张被明确拒绝。
- 在视频模式导入 100 MB 内、8 秒内片段的视频时可预览和导出；超出 100 MB、8 秒或 240 帧预算时被明确拒绝。
- 多图可追加、排序和删除，预览与导出顺序一致。
- 视频片段开始/结束时间调整后，预览与导出使用同一片段范围。
- 三种转场均可循环预览和导出，最后一张到第一张无错误闪帧。
- 横竖比例混合图片在统一画布中不拉伸、不裁剪，空白区颜色一致。
- 删除至 1 张后多图导出被禁止但素材可继续追加恢复；切回单图后单图功能继续可用。
- 所有失败路径保留可恢复状态，图片、视频和生成结果不发生网络上传。
- `npm test`、`npm run typecheck`、`npm run lint`、`npm run build` 通过，并完成目标浏览器手工验收。

## 9. 明确不做

- 不支持音频、文字、图层、蒙版、滤镜或远程图片/视频 URL。
- 不提供时间轴、自由关键帧、逐张时长、逐段转场或多轨编辑。
- 不在多图模式叠加六种单图动效。
- 不引入 `ffmpeg.wasm`、服务端转码、多视频拼接或音频导出。
- 不提供账号、云存储、跨设备同步、自动保存或服务端渲染。
- 不导出 MP4、WebM、APNG 或项目工程文件。
- 不进行移动端专项优化，也不保证超出约定上限的素材可处理。
